## v0.1.1

修复 Linux 一键安装后默认 SQLite 配置无法启动的问题。

### 本次修复

- 将 SQLite 驱动切换为纯 Go 实现，避免发布二进制依赖 CGO
- 修复 Debian 12 等环境下一键安装后服务启动失败的问题
- 保持当前 GitHub Release 交叉编译和单文件发布方式不变
- 默认 SQLite 部署现在可以直接使用官方 Release 包启动

### 影响范围

- 使用默认 SQLite 配置的部署场景
- 通过 GitHub Release 下载预编译二进制的用户
- Linux 一键安装用户

### 升级说明

如果你在 `v0.1.0` 中遇到以下错误：

`go-sqlite3 requires cgo to work`

请升级到 `v0.1.1`。
