# v0.1.13

本次版本聚焦订阅信息页的"备用订阅链接"体验，修复 Clash 系客户端导入时被强制喂入 V2Ray Base64 内容导致 `invalid yaml` 报错的问题。

## 亮点

- 订阅信息页"备用订阅链接"卡片由单条改为按客户端类型分组展示：通用 Base64（V2RayN / NekoRay / Shadowrocket）、Clash 系（Mihomo / Clash Verge / ClashX）、Sing-box（sing-box / Hiddify），每条都带客户端示例提示与复制 / 二维码按钮。
- 备用链接统一改用显式的 `?format=v2ray|clash|singbox` 参数，与 UA 自动识别解耦，复制后即贴即用，避免客户端 UA 异常时回退到错误格式。
- 主订阅链接（`/sub/<token>`）行为不变，仍按 `User-Agent` 智能识别 Clash / Sing-box / V2Ray，并维持向后兼容的 `?sub=1` 别名。

## 适合升级的场景

- Clash Verge / Mihomo 用户直接复制"备用订阅链接"导入失败，提示 `the remote profile data is invalid yaml`。
- 客户端 User-Agent 被网络中间件或代理改写后，主订阅链接无法被正确识别为 Clash 格式。
- 希望让不同客户端用户在订阅页一眼看到自己该用的链接，减少前台咨询。
