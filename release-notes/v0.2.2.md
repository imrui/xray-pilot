# xray-pilot v0.2.2

本次版本补齐 SQLite 数据库定时备份与备份文件管理界面，作为灾备底线。备份通过 `VACUUM INTO` 生成完整且一致的二进制快照，对运行中的服务不产生写阻塞。

## 亮点

- 新增 `BackupService`，每 24 小时（可配）自动生成 `xray-pilot-YYYYMMDD-HHMMSS.db` 完整快照到 `data/backup/`。
- 备份文件自动按保留天数清理（默认 30 天），无需运维介入。
- 系统设置页新增「数据备份」面板：列出所有备份（文件名、大小、创建时间）、立即备份按钮、单个备份的下载/删除操作。
- 备份目录、调度间隔、保留天数全部可在 UI 修改并即时生效。
- 备份文件名路径校验严格，下载/删除接口拒绝 `../` 等 path traversal 尝试。

## 后台变更

- 新增配置项：
  - `backup.dir` — 备份目录（默认 `data/backup`）
  - `backup.interval_hours` — 调度间隔（默认 `24`，设 `0` 禁用）
  - `backup.retention_days` — 保留天数（默认 `30`）
- 新增 HTTP API：
  - `GET /api/system/backups` 列表
  - `POST /api/system/backups` 立即备份
  - `GET /api/system/backups/:name/download` 下载
  - `DELETE /api/system/backups/:name` 删除
- 新增定时任务 `backup_run`，启动后 1 小时延迟首次执行，避免与启动期其他任务争抢资源。

## 注意事项

- **仅支持 SQLite**：Postgres 用户请继续使用 `pg_dump`，本次未在面板内提供 Postgres 备份能力（YAGNI）。
- **本地存储**：备份与服务进程同主机，主机磁盘损毁会同时丢失业务数据与备份。**强烈建议自行通过 cron/rsync/对象存储客户端**把 `data/backup/` 定期拉到异地。
- **不含自动恢复**：恢复需停服 → 替换 `xray-pilot.db` 文件 → 重启进程，本次未提供 UI 上传恢复（避免误操作炸库）。下载的 `.db` 文件可直接放回原 DSN 路径使用。
- **MasterKey 仍需单独保管**：备份只是 db 文件本身，Reality 私钥仍以 AES-GCM 加密形式存储；丢失 MasterKey 后即便有 db 备份也无法解出私钥。建议把 MasterKey 离线另存一份。

## 适合升级的场景

- 主机重装/换机/升级 Go 版本前希望先有一份可回滚的数据快照。
- 担心 SQLite 文件损坏（磁盘异常、断电），需要每天滚动备份。
- 想保留一段历史窗口便于回溯"某天为什么用户列表变了"。
