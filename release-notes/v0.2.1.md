# xray-pilot v0.2.1

接入 xray-core gRPC StatsService，实现按用户维度的流量统计采集与展示。

## 新增

- **定时采集**：每 5 分钟（可配 `scheduler.traffic_poll_interval`）通过 SSH 隧道访问远端节点 xray gRPC API，拉取并清零计数后落库
- **用户列表「累计流量」列**：展示 ↑ / ↓ 字节人类可读单位
- **仪表盘流量卡片**：新增「累计上行」「累计下行」「近 7 天活跃用户」三张卡片，每分钟自动刷新
- **API**：`GET /api/traffic/summary` 供仪表盘消费；`GET /api/users` 响应携带 `traffic_up_bytes` / `traffic_down_bytes` / `traffic_last_updated_at`

## ⚠️ 升级后必须做的事

xray 服务端默认不开启流量统计。本版本已在配置生成中补齐 `stats` 与 `policy` 段，**升级后需要在管理面板手动触发「同步所有节点」一次**，让新配置下发并 reload，统计才会真正开始累积。同步前 Dashboard 与 Users 流量字段会停留在 0。

## 不在本版本范围

- 配额上限与超额自动禁用
- 按周 / 月聚合视图与流量趋势图
- 节点维度的流量明细页
