# xray-pilot v0.2.1

本次版本接入 xray-core gRPC StatsService，实现按用户/节点维度的流量统计采集，并在管理面板里以「累计上/下行」列与仪表盘卡片形式呈现。本版聚焦最小闭环（采集 → 持久化 → 展示），不含配额限制与超额自动禁用。

## 亮点

- 新增 `TrafficPoller` 定时任务，每 5 分钟（可通过 `scheduler.traffic_poll_interval` 配置）通过 SSH 隧道访问远端节点 `127.0.0.1:10085` 的 xray gRPC API，调用 `StatsService.QueryStats(reset=true)` 拉取并清零计数，把增量字节落库。
- 数据双表设计：`traffic_samples` 保存 5 分钟粒度的明细，便于后续做趋势/退订分析；`user_traffic_totals` 记录每用户跨节点累计，供 Users 列表与 Dashboard 直接消费。
- Users 表格新增「累计流量」列，展示 ↑ / ↓ 字节人类可读单位；Dashboard 增加「累计上行」「累计下行」「近 7 天活跃用户」三张卡片，每分钟自动刷新。
- 新增 `GET /api/traffic/summary` 接口供仪表盘消费；`GET /api/users` 响应携带 `traffic_up_bytes` / `traffic_down_bytes` / `traffic_last_updated_at` 字段。
- 不引入 xray-core 依赖：在 `internal/xray/statspb/` vendor 了最小化 `command.proto` 并生成 Go gRPC stub，编译产物仍保持 ~30MB 量级。

## ⚠️ 升级后必须做的事

xray 服务端默认不开启流量统计，必须在配置里显式声明 `stats` 模块与 `policy` 段。本版本已在 `internal/xray/config.go` 补齐这两段，**升级后需要在管理面板手动触发「同步所有节点」一次**，让新生成的 xray 配置下发到每台节点并 reload，统计才会真正开始累积。同步前 Dashboard 与 Users 流量字段会一直停留在 0。

## 实现要点 / 已避坑

- **Reset 语义**：`QueryStats(reset=true)` 是"先取后清零"，返回值即周期增量，无需差分；xray 进程崩溃会导致内存计数器丢失，本版本接受这种偏差。
- **SSH 隧道**：通过 `golang.org/x/crypto/ssh` 的 `client.Dial` 把远端 loopback gRPC 转出，gRPC 客户端使用 `passthrough:///` resolver 绕开默认 DNS 解析，避免 target 被改写。
- **失败处理**：单节点连接/查询失败时跳过本周期、不补 0，避免误报为"0 流量"；连续失败的节点会在 SyncLog 中记录便于排查。
- **orphan 计数**：xray 上残留的已删除用户 email 计数会被丢弃；由于同时已被 reset 清零，下次轮询不会重复出现。

## 不在本版本范围

以下能力暂未包含，将在后续版本评估：
- 配额上限设置与超额自动禁用
- 按周/月维度的聚合视图与流量趋势图
- 节点维度的流量明细页
- `traffic_samples` 表的自动 retention（当前需手动清理）

## 适合升级的场景

- 希望在管理面板查看每个订阅用户实际跑了多少流量，识别异常用量。
- 想了解节点整体被使用情况，作为后续扩容/下线决策依据。
