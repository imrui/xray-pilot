# xray-pilot v0.2.3

本次版本在订阅信息页（用户访问 `/sub/<token>` 看到的 HTML 页）的「用户信息」卡片下方补充累计流量展示，用户无需登录管理面板即可自助查看自己当月的用量。

## 亮点

- 「用户信息」卡片底部新增一节「累计流量」，两列并排显示 ↑上行 / ↓下行，下方补「合计」与「最近更新」时间。
- 从未产生流量的新用户显示「尚无流量记录」占位，避免卡片留空显得未完成。
- 字节格式化使用二进制单位（KiB / MiB / GiB），与 Dashboard 行为一致；数字使用等宽字体（`tabular-nums`），多用户对比时数字位置稳定。
- 信息页布局保持不变（左卡片下半部分原本留白），未挤压右侧二维码区域。

## 实现要点

- 后端 `SubscribePageData` 同时输出原始 `int64` 字节与预格式化的 `string`，便于将来 API 复用（不强迫 HTML 模板做字符串组装）。
- 流量数据来自 `UserTrafficTotal`（v0.2.1 引入），查询失败时静默回退到「尚无流量记录」，信息页不会因辅助信息失败而 5xx。
- 后端 `formatBytes` 与前端 `frontend/src/lib/utils.ts` 行为对齐，避免前后端格式分裂。

## 适合升级的场景

- 想减少 "我这个月用了多少？" "我的流量统计在哪看？" 等用户咨询。
- 希望订阅信息页本身就具备基本自助查询能力，不必引导用户登录管理面板。
- 用户跑流量后想立即得到正向反馈，确认订阅与节点工作正常。
