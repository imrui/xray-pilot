  # xray-pilot v0.2.0

本次版本补齐订阅模块对 sing-box 客户端的原生支持：`?format=singbox` 现在返回真正的 sing-box JSON 配置文件，而非过去回退使用的 V2Ray Base64 URI 列表。Hiddify、Karing、NekoBox、sing-box GUI 等以 sing-box 为内核的客户端从此可直接导入订阅获得 selector / urltest 代理组体验，无需再借助通用 Base64 链接手动转换。

## 亮点

- 订阅 `?format=singbox` 端点输出完整可用的 sing-box JSON Profile，包含 `log` / `dns` / `inbounds`（tun + mixed）/ `outbounds` / `route` / `experimental.clash_api`，导入即可使用。
- 节点 outbound 自动包装为 `selector`（手动切换）+ `urltest`（自动择优，3 分钟测延迟）组合，对应 GUI 客户端的"代理组"UI。
- 完整覆盖现有 4 种协议的 sing-box outbound 渲染：VLESS+REALITY+Vision、VLESS+WS+TLS、Trojan、Hysteria2。
- 响应头按 sing-box 社区惯例调整：`Content-Type: application/json`、`Content-Disposition: attachment; filename="xray-pilot.json"`；同时 Clash 订阅文件名也补全为 `xray-pilot.yaml`。
- 基线 sing-box 1.8.0，覆盖 Hiddify ≥2.0 / Karing ≥1.0 / NekoBox ≥1.2.7；不引入 rule_set 远程依赖，避免首次启动需联网下载 geosite / geoip 拖累体验。

## 适合升级的场景

- 使用 Hiddify、Karing、NekoBox、sing-box GUI 等 sing-box 内核客户端的成员，希望直接导入订阅链接而非手动配置。
- 客户端导入"备用订阅链接 - Sing-box"卡片后无可用代理组，或仅能识别一个节点。
- 跨平台团队（iOS Hiddify + Android NekoBox + Desktop sing-box）希望共用同一份订阅来源，并在客户端 GUI 中享受统一的代理选择 / 自动测速行为。

## 升级与兼容性

- 升级无需任何配置变更，订阅页"备用订阅链接 - Sing-box"卡片继续可用，链接形态不变。
- V2Ray Base64（`?format=v2ray`）与 Clash YAML（`?format=clash`）输出保持原状，未做改动。
- 当用户未绑定任何分组或绑定分组下无可用节点时，sing-box 订阅返回最小骨架（仅含 direct / block / dns 出站），客户端导入不会报错。
