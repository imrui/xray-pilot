## v0.1.2

本版本主要完善了部署可用性、诊断能力，以及反向代理与订阅生成相关体验。

### 本次更新

- 新增部署诊断能力
  - 后台系统设置页可直接检查数据库目录、默认 SSH 私钥、`known_hosts`、订阅基址和配置文件可读写状态
  - 为常见部署问题提供直接的修复建议

- 优化 Linux systemd 部署默认值
  - 默认 SSH 私钥路径调整为 `/etc/xray-pilot/ssh/id_ed25519`
  - 默认 `known_hosts` 路径调整为 `/var/lib/xray-pilot/known_hosts`
  - 安装脚本会自动创建对应目录和文件，减少首次部署权限问题

- 修复反向代理场景下的订阅地址识别
  - 订阅链接现在支持识别 `X-Forwarded-Proto`、`X-Forwarded-Host` 等代理头
  - 在 Nginx / HTTPS 反代环境下，可正确生成 `https://` 订阅地址

- 修复订阅备注编码问题
  - 节点备注不再出现 `+` 代替空格的显示效果

- 完善设置页
  - 增加 `ssh.default_key_path`
  - 增加 `ssh.known_hosts_path`
  - 增加部署建议与排障信息展示

### 升级建议

如果你已经通过安装脚本部署旧版本，升级到本版本后建议检查：

- `subscription.base_url` 是否已配置为公网 HTTPS 地址
- 默认 SSH 私钥是否已放置在 `/etc/xray-pilot/ssh/id_ed25519`
- `known_hosts` 是否位于 `/var/lib/xray-pilot/known_hosts`

必要时可重新执行安装脚本完成升级：

```bash
curl -fsSL https://raw.githubusercontent.com/imrui/xray-pilot/main/install.sh | sudo bash
```
