# xray-pilot v0.1.2

完善部署可用性与反向代理场景下的订阅生成。

## 新增

- **部署诊断**：系统设置页可一键检查数据库目录、SSH 私钥、`known_hosts`、订阅基址与配置文件的可读写状态，并给出修复建议
- **systemd 默认路径调整**：SSH 私钥默认 `/etc/xray-pilot/ssh/id_ed25519`、`known_hosts` 默认 `/var/lib/xray-pilot/known_hosts`，安装脚本自动创建对应目录

## 修复

- 反向代理场景下订阅地址识别 `X-Forwarded-Proto` / `X-Forwarded-Host`，Nginx HTTPS 反代后能正确生成 `https://` 订阅地址
- 节点备注不再出现 `+` 代替空格的编码问题

## 升级

如已用旧版安装脚本部署，建议检查 `subscription.base_url` 是否配为公网 HTTPS 地址，必要时重跑安装脚本。
