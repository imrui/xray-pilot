# xray-pilot v0.3.0

多协议共存能力 + 节点协议双向编辑。

## 新增

### 单节点多协议安全共存

同一节点可同时绑定 VLESS+Reality / Trojan / VLESS+WS+TLS 等多个协议，互不冲突。每个协议可单独覆盖监听端口（如 Reality 占 443，同节点 Trojan 挪到 8443）；后端按 TCP / UDP 分别校验端口唯一性，冲突时直接拒绝并给中文错误。订阅 URI、Clash、sing-box 三种输出格式都跟随节点端口生成，客户端连接端口与节点实际监听一致。

### 协议轴抽屉：批量为多节点配置同一协议

「配置节点密钥」抽屉重写为双栏布局。左侧多选节点 → 右侧批量生成密钥 / 批量保存。例如 12 节点全 Reality 场景，三步内（全选 → 批量生成 → 批量保存）即可完成，每节点仍生成独立密钥。

### 节点轴抽屉：批量为单节点绑定多协议

节点列表点击协议徽章打开新的「节点协议管理」抽屉。左侧多选协议 → 右侧批量绑定，新节点接入时一次完成所有协议挂载。编辑时端口冲突会实时提示并阻断保存。

### 节点所有者属性

节点新增「所有者」字段，标识来源（供应商代号、团队名等）。节点列表新增所有者列与筛选下拉，搜索覆盖该字段。

## 关于 Hysteria2

Hysteria2 是 Xray 之外的独立项目，xray-core 内核不识别该协议。本版本：

- 订阅输出（URI / Clash / sing-box）**完整支持** Hysteria2，且端口冲突护栏正确把它标记为 UDP，可与 Reality 在 443 端口共存
- Xray 配置生成层**不**为 Hysteria2 生成 inbound，避免让节点的 xray 启动失败

真正的 Hysteria2 服务端管理（独立 hysteria 二进制部署管线）会作为独立功能在后续版本评估。
