# xray-pilot v0.3.0

本次版本完成节点与协议关系的双向重构：协议轴抽屉支持多节点批量生成密钥，新增镜像的"节点协议管理"抽屉以节点为目标管理多协议；同步建立节点级端口覆盖与端口冲突护栏，让 VLESS+Reality / Trojan+TLS / VLESS+WS+TLS 真正能在同一节点上安全共存。同时为节点新增"所有者"属性，标识节点来源。

## 亮点

### 单节点多协议安全共存（地基）

- **节点级端口覆盖**：`NodeProfileKey` 新增 `Port` 字段，节点可在不污染协议模板的前提下，把某个协议的监听端口改到非默认值（例如 Reality 占 TCP 443，同节点的 Trojan 挪到 8443）。GORM AutoMigrate 自动建列，零迁移脚本。
- **端口冲突护栏**：保存节点密钥时按 `(传输层, 有效端口)` 校验同节点内唯一性——TCP 与 UDP 各自独立编址，互不冲突；Hysteria2 标记为 UDP，可与 Reality 在 443 端口共存。冲突时 service 层硬拒并返回中文错误；`xray.GenerateConfig` 同时做兜底拦截，防御旧数据或手工改库导致 Xray 启动失败。
- **订阅链路全部跟随节点端口**：URI / Clash / sing-box 三套客户端导出原先都直接读 `profile.Port`，节点改了端口客户端会连不上。现在统一走 `effectiveKeyPort()`，节点 Xray 实际监听端口与客户端订阅看到的端口完全一致。

### 协议轴抽屉：单协议 → 多节点批量

把"配置节点密钥"抽屉重写为左右双栏布局：

- **左侧节点列表**：节点搜索、全选可见、复选框多选、激活节点高亮；每项实时展示「已绑/未绑、端口、🔒 锁定、未保存草稿点」状态。
- **右侧编辑区**：单选节点时可编辑 settings JSON、端口覆盖、锁定/解锁；多选节点时进入批量模式，支持"批量生成密钥（每节点独立 keygen）"与"批量保存"，逐节点反馈成功/失败。
- **解决的痛点**：12 节点全 Reality 的场景，原先要切 12 次下拉、点 12 次生成，现在「全选可见 → 批量生成 (12) → 批量保存 (12)」三步搞定，且每节点仍生成独立密钥。

### 节点轴抽屉：单节点 → 多协议管理（新）

节点列表点击协议徽章弹的旧 Modal 升级为完整的协议管理抽屉，与协议轴抽屉**镜像对称**：

- **左侧协议清单**：列出所有协议，标注已绑/未绑、端口覆盖、🔒 状态；支持搜索、多选、激活。
- **右侧编辑区**：单选时可编辑 settings + 端口 + 一键生成 + 锁定/解锁 + **移除协议**；多选时支持"批量绑定"——勾选 N 个未绑协议，一次性以模板默认参数完成绑定，再逐个微调。
- **端口冲突前端实时提示**：基于已绑协议的 `传输层+有效端口` 实时检测，提示"将被后端拒绝"并 disable 保存按钮，提前阻断错误。
- **新节点初始化场景**：原先需要在协议页切 N 次协议、找新节点配；现在直接打开节点抽屉、全选可见、批量绑定，一次完成全部协议接入。

### 节点所有者属性

- `Node` 新增 `Owner` 字段，标识节点来源（如供应商代号、内部团队）。
- 节点列表新增「所有者」列、独立筛选下拉、关键字搜索覆盖；分组编辑的节点选择列表也补展所有者徽标与域名信息。
- 节点协议抽屉的信息卡显示「地区 / 所有者」，空值用 `--` 占位（与列表一致），带字段标签防止混淆。

## 体验细节

- **协议徽章配色统一**：协议轴列表、节点轴列表、节点协议抽屉左列、批量预览四处协议徽章全部用同一套配色（Reality 绿 / WS+TLS 蓝 / Trojan 黄 / Hysteria2 红）。所有展示元数据收敛到 `frontend/src/lib/protocol.ts` 单一来源。
- **协议列徽章可点击**：节点列表的协议单元格里，第一个协议徽章直接点击即可打开节点协议抽屉（hover 浮起 + focus ring 提示可点）；冗余的"密钥/锁定"按钮保留作为次级入口。
- **Drawer 组件新增 `xl` 宽度档（960px）**：满足节点协议抽屉的双栏布局需求；保持 `md/lg` 后向兼容。
- **节点协议抽屉的节点信息卡** 始终显示「地区 -- · 所有者 --」格式，带字段标签 + `--` 占位，单字段为空时不再产生"这是地区还是所有者"的歧义。

## 关于 Hysteria2 的诚实交付

Hysteria2 是独立项目（apernet/hysteria），**Xray-core 不识别该协议类型**——在 `xray/config.go` 里硬塞 hysteria2 inbound 会让 Xray 整个起不来。本版本：

- ✅ 端口护栏正确把 Hysteria2 标记为 UDP，**与 Reality TCP 443 不冲突**，可作为同节点的第二协议保存与订阅输出
- ✅ 订阅链路（URI / Clash / sing-box）对 Hysteria2 输出完整支持，使用节点级端口覆盖
- ❌ 不在 Xray 配置里生成假的 hysteria2 inbound，避免给一个跑不通的功能误导用户

真正的 Hysteria2 服务端管理（独立 hysteria 二进制 + systemd + server.yaml 下发）是与 Xray 平行的另一套部署管线，会作为独立功能在后续版本评估。

## 实现要点

- **共享前端 lib**：抽出 `frontend/src/lib/keygen.ts`（Reality short_id 客户端生成）与 `frontend/src/lib/protocol.ts`（协议标签/配色/选项映射），消除两个抽屉之间的重复代码。
- **冲突检测语义对齐**：`pkg/types/ProtocolTransport()` 是端口冲突判定的单一来源；service 层在保存前校验，xray 配置生成在循环里做兜底；前端抽屉在编辑期实时检测。三处对齐。
- **回归保障**：新增 `TestEffectiveKeyPort` 单测覆盖四种端口回退分支，固化"订阅输出与节点 Xray 监听端口一致"这个核心不变量。
- **schema 变更**：`node_profile_keys` 表新增 `port` 列（默认 0 = 继承），GORM AutoMigrate 启动时自动添加，老数据零侵入。

## 适合升级的场景

- 想在同一节点上同时跑 Reality + Trojan / WS+TLS，做线路冗余
- 单 Reality 多节点（如 12 节点同款配置）覆盖场景，希望批量生成密钥
- 新节点接入需要一次性绑定多个协议
- 需要给节点打"来源标签"（哪个供应商、哪个团队），方便筛选与审计
