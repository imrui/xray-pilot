# xray-pilot v0.4.3

修复"删节点后新建节点会继承旧节点的分组、协议、用户关联"的数据串问题。

## 修复

### 节点删除时级联清理关联表

之前删节点只 `DELETE FROM nodes`，但 `group_nodes`（节点-分组）、`node_profile_keys`（节点-协议密钥）两张关联表的对应行未清。叠加 SQLite 删除最大 ID 后下次 INSERT 复用 ID 的机制——一键接入新建节点正好撞到旧节点 ID 时，新节点会"继承"旧分组、旧协议、旧用户。

现在删节点改为事务，三张表一起删：`group_nodes` → `node_profile_keys` → `nodes`。

### 启动时一次性清理历史孤儿关联

升级到 v0.4.3 后首次启动会跑一次孤儿清理，覆盖六个方向：
- `group_nodes` 指向已不存在的节点 / 分组
- `node_profile_keys` 指向已不存在的节点 / 协议
- `user_groups` 指向已不存在的用户 / 分组

之前历史残留的脏数据会一并清掉。**清理动作只针对孤儿行，不影响合法关联**。

## 升级

- 无 schema 变更，无破坏性 API 变更
- 启动后建议复查一下节点列表里近期一键接入的节点，确认分组、协议、所有者字段都正确（若仍有残留可手工修正后下次启动会自动清干净）

## 注意事项

- 本版本未改为软删（GORM `DeletedAt`），ID 复用仍可能发生。但有了级联清理和启动 cleanup 后，复用 ID 不再带来数据串
- 软删改造涉及面较大（所有节点查询要带 `WHERE deleted_at IS NULL`、unique 约束改 partial index），暂不在本版本范围
