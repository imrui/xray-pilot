# xray-pilot v0.3.1

三处前端修复：暗色主题下原生 select 选项不可见、节点密钥抽屉的列表高度与节点项布局、备份文件下载因缺失 JWT 被拒为 401。

## 修复

- **暗色主题下原生 `<select>` 下拉选项不再白底白字**：浏览器原生弹层并不总是遵循 `color-scheme`，部分 chromium/edge 版本下选项会沿用系统亮色背景。全局给 `option` 显式声明 `background-color: var(--panel-strong)` + `color: var(--text)`，所有页面的筛选下拉、分页选择、表单 SelectField 一次性修复。
- **配置节点密钥抽屉：节点列表高度自适应抽屉**：原先 `max-h-[560px]` 硬限高，与 Drawer 实际可视高度脱节，会在还没到底部时提前出现滚动条且抽屉底部留出大片空白。改为 flex 自适应布局：grid 容器 `h-full`、左侧列 `flex h-full min-h-0 flex-col` + 列表区 `flex-1 overflow-y-auto`、右侧编辑区独立 `min-h-0 overflow-y-auto` 滚动，列表始终占满抽屉可用高度。
- **节点列表项 IP 过长不再遮盖端口与 🔒 锁图标**：原先 IP / 端口 / 锁图标共用同一个 `truncate` span，IP 长时会把后两者一起截掉。改为 IP `flex-1 truncate`、"· 端口 X 🔒" 包在 `shrink-0` 容器里始终可见；端口与协议模板默认值一致时显示"默认"以节省字符宽度。
- **备份文件下载：恢复 JWT 鉴权**：之前用 `<a href download>` 触发浏览器原生导航，**不会带 axios 注入的 `Authorization: Bearer ...` header**，被后端 JWT 中间件拒为 401，浏览器把错误响应体当成附件存成 `download.json`。改用 axios `responseType: 'blob'` 获取数据，从 `Content-Disposition` 解析后端给的真实文件名（`xray-pilot-YYYYMMDD-HHMMSS.db`），再用 `URL.createObjectURL` 触发浏览器另存。

## 实现要点

- `backupApi.downloadURL(name)` → `backupApi.download(name)`：从「返回 URL 字符串」改为「异步触发下载」。Settings 页对应入口从 `<a>` 改为 `<button>`，下载失败时统一通过 toast 提示。
- 后端 `c.FileAttachment` 已正确设置 `Content-Disposition: attachment; filename="..."`，本次无需改动后端。
- option 全局样式置于 `@layer base`，被组件内 utilities 覆盖时仍可被复用。

## 升级建议

纯前端修复，**无 schema 变更、无后端逻辑变化**，可平滑覆盖升级。
